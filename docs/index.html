TDD Learning Methodology

Based on the proven "Learn Go with Tests" approach

‚ùå

1. Write Test First (Red)

Start with a failing test that defines the behavior you want. This helps you think about the API design before implementation.

func TestLinuxSocket(t *testing.T) {
    socket := NewLinuxSocket()
    err := socket.Send([]byte("test"))
    // This will fail - we haven't implemented it yet!
}
‚úÖ

2. Make It Pass (Green)

Write the minimal code to make the test pass. Don't worry about perfect code - just make it work.

func (s *LinuxSocket) Send(data []byte) error {
    _, err := syscall.Sendto(s.fd, data, 0, s.addr)
    return err
}
üîÑ

3. Refactor (Blue)

Now improve the code while keeping tests green. Add error handling, optimize performance, improve readability.

func (s *LinuxSocket) Send(data []byte) error {
    if len(data) == 0 {
        return errors.New("empty data")
    }
    n, err := syscall.Sendto(s.fd, data, 0, s.addr)
    if err != nil {
        return fmt.Errorf("send failed: %v", err)
    }
    if n != len(data) {
        return errors.New("partial send")
    }
    return nil
}
